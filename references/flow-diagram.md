## AgentForge Skill 交付物总览

| 交付物 | 说明 |
|--------|------|
| **System Prompt** | 可直接部署的 Skill 核心指令 |
| **对话流程** | 10 轮引导 + 迭代轮 |
| **7 个 MD 模板** | 人类可读的完整文档体系 |
| **blueprint.yaml** | 机器可读的完整执行蓝图 |
| **3 个执行 Prompt** | Phase 1 / Phase 2 / 迭代更新 |

## 使用说明

### 第一步：触发 AgentForge Skill

在 Claude Code 中，当你提到以下任何关键词时，AgentForge skill 会自动触发：
- "创建 AI Agent"
- "搭建智能体"
- "构建 AI 助手"
- "设计 Agent 架构"
- "生成项目蓝图"
- "开发聊天机器人"
- "做自动化工具"
- 或任何涉及 LLM/大模型的应用

### 第二步：多轮对话收集需求

AgentForge 会通过 9 轮专业对话，逐步收集：
1. 核心功能需求
2. 使用场景和用户画像
3. 产出形式（GUI/CLI）和目标平台
4. 技术栈选型
5. AI 模型配置
6. 工具与外部集成
7. 项目复杂度评估
8. 特殊需求（性能/安全/UI）
9. 最终汇总确认

### 第三步：生成完整蓝图

确认无误后，AgentForge 会生成：
- **7 个 Markdown 文件**（人类可读）：
  - README.md
  - REQUIREMENTS.md
  - ARCHITECTURE.md
  - UI_DESIGN_GUIDE.md（GUI 项目）
  - API_INTEGRATION.md
  - TODO.md
  - TESTING.md
- **1 个 YAML 文件**（机器可读）：
  - blueprint.yaml

### 第四步：交给 AI IDE 执行

将 `blueprint.yaml` 交给 AI IDE（Cursor / Windsurf / Cline），它会：
1. **Phase 1**：执行项目初始化（仅一次）
2. **Phase 2**：按照增量循环模式开发所有功能

### 第五步：跟踪进度

通过 `TODO.md` 实时查看开发进度：
- `[Todo]` — 待开发
- `[In Progress]` — 开发中
- `[Done]` — 已完成
- `[Failed]` — 开发失败（详见 FAILURES.md）

### 第六步：迭代更新（可选）

如果需要修改需求，随时告诉 AgentForge，它会：
1. 分析变更影响
2. 增量更新所有蓝图文件
3. 生成新版本的 blueprint.yaml
4. 你可以继续交给 AI IDE 执行更新

## 核心优势

### 1. 外部记忆架构
- 每轮 AI 对话上下文可清零
- 文件系统（Git）永远保留完整状态
- 通过读取文件恢复上下文，无限扩展开发能力

### 2. 双阶段开发
- **Phase 1**：一次性初始化，搭建完整脚手架
- **Phase 2**：增量循环开发，每轮只做一个功能，TDD 驱动

### 3. 失败容错
- 单个功能最多重试 3 次
- 失败后自动回滚，记录到 FAILURES.md
- 跳过失败功能，继续开发其他功能

### 4. UI 一致性保证
- GUI 项目自动生成 UI_DESIGN_GUIDE.md
- 每轮开发前强制读取设计规范
- 确保全局视觉风格统一

### 5. 模型适配层
- 通用适配器模式
- 支持零代码切换 AI 模型供应商
- 环境变量配置，灵活切换

## 常见问题

### Q1: 我可以中途修改需求吗？
**A**: 可以。随时告诉 AgentForge 你的变更需求，它会增量更新蓝图。

### Q2: 如果某个功能开发失败怎么办？
**A**: Phase 2 会自动重试 3 次。3 次全失败后，记录到 FAILURES.md，跳过继续下一个功能。最后会汇总展示所有失败项。

### Q3: 我可以自己修改生成的蓝图文件吗？
**A**: 可以。所有文件都是标准格式，你可以手动编辑。修改后重新交给 AI IDE 执行即可。

### Q4: 支持哪些 AI 模型？
**A**: 支持所有主流模型：OpenAI GPT、Anthropic Claude、Google Gemini、DeepSeek 等。通过通用适配层，可随时切换。

### Q5: 生成的代码质量如何保证？
**A**: 采用 TDD（测试驱动开发）方法，每个功能先写测试，测试通过才提交。确保代码质量。

### Q6: 我可以用于商业项目吗？
**A**: 可以。生成的所有代码和蓝图文件归你所有，无任何使用限制。
